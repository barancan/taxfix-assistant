# Dependency-age policy

**No npm package version released within the previous 10 calendar days may be
used** — direct, dev, or transitive. Versions are pinned exactly (no `^`/`~`).

## How it works
- `scripts/audit-package-age.mjs` reads `package-lock.json`, enumerates every
  resolved `name@version`, queries the npm registry `time` metadata, and **fails**
  if any is < 10 days old. It **fails closed** when publish metadata cannot be
  verified (unless listed as a documented manual exception — currently none).
- Run: `npm run audit:package-age` (add `--write-report` to emit
  `docs/dependency-audit.report.json`). Re-run before every milestone merge and
  the final build.

## Overrides
Several "latest" versions were too new at install and are pinned to older eligible
versions via `overrides` in `package.json` (typescript-eslint family, the
browserslist/postcss build chain, vite, etc.). The audit confirms the resolved
tree (555+ packages) is fully compliant.
